# E3.7b — Logout reachable at all viewport widths (fix 1024–1279px gap)

## Problem
The designer topbar (`server/templates/designer.html:23-64`) is a fixed-height,
non-wrapping `display:flex` row; `.topbar-right` packs eight controls incl. `#logout-btn`.
The burger overflow menu (into which logout is reparented) only activates at
`max-width: 1023.98px` (`responsive.css:27`, `responsive-layout.js:30` `mqTablet`,
`responsive-layout.js:52-53` `currentMode()` desktop cutoff). There is **no** media rule
above `1023.98px`. Between **1024px and ~1279px** the eight-control cluster exceeds the
available width and clips, and because the burger is inactive there, `#logout-btn` is
**not reachable** — an authenticated user cannot log out in that band.

## Acceptance criteria
- **AC1 — Logout always reachable.** For an authenticated user, the Logout control is
  present and clickable at EVERY viewport width 320–1920px. Explicitly verify the
  previously-broken band: **1024, 1100, 1200, 1279px** (logout visible OR reachable via
  the burger, and a click actually triggers logout).
- **AC2 — Minimal breakpoint fix.** Extend the compact/burger-overflow handling upward to
  the width at which the desktop single-row topbar actually fits without clipping
  (determine empirically headless; expected ≈1280px). Update the SAME value in all three
  coupled spots so they stay consistent: `responsive.css:27` media query, the `mqTablet`
  upper bound (`responsive-layout.js:30`), and the `currentMode()` desktop cutoff
  (`responsive-layout.js:52-53`). Below the new cutoff, controls (incl. logout) live in the
  burger as they already do for tablet.
- **AC3 — No clipping / no body scroll.** No topbar control is clipped or overflowing at
  any width; the page body never scrolls horizontally because of the topbar.
- **AC4 — Desktop unchanged.** At widths ≥ the chosen fit-width the roomy single-row
  desktop layout is visually identical to before. Mobile (<768) and tablet (768–1023)
  behavior is unchanged. Only the 1024–(fit-width) band changes behavior.
- **AC5 — Auth states.** Logout is shown only when authenticated (`auth.js`
  `showLogoutButton` at `auth.js:242-253` intact). The reparenting must NOT make logout
  appear for anonymous users at any width.
- **AC6 — Headless evidence.** Puppeteer/CDP sweep across the widths in AC1 with captured
  evidence that logout is present + clickable in the broken band and nothing clips. Zero
  JS console errors introduced.

## Non-goals
- No topbar redesign, no new overflow component — minimal breakpoint change only.
- Not fixing the separate documented 1024–1594 theme/settings-truncation cosmetics beyond
  ensuring nothing is *unreachable*.
