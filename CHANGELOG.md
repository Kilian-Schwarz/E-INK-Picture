## [Unreleased]

### Added

- Guided setup wizard for fresh installations (E2.3): opening `/designer` on a factory-fresh system (no admin password, no `setup_completed` latch, no user-touched settings keys, only the untouched default design, no history snapshots, empty media library — a client heartbeat alone does NOT count as usage) now shows a full-screen, mobile-first five-step wizard instead of the empty canvas: (1) display type as two cards with a schematic panel illustration (inline SVG, color stripes from the driver palette) plus a server-time/timezone sanity line, (2) optional weather location via the existing Nominatim proxy `GET /location_search` (search only on click/Enter, ≥ 1 s between requests, coordinates trimmed to 4 decimals), (3) refresh interval presets 15/30/60 min with a panel-lifetime hint and an Advanced render-quality select — saved as exactly ONE `POST /update_settings` before the gallery renders, (4) mandatory admin password (`POST /api/auth/setup` + immediate auto-login, same sequence as the E5.1 banner dialog; 403 on a two-browser race links to `/login`), (5) starter design from the template gallery (panel-true previews; picking a card applies the E3.5 instantiation pass plus a wizard-only location pass — weather/forecast widgets get the chosen coordinates, the weather-dashboard's "BERLIN" label becomes the chosen place name — then creates, ACTIVATES and triggers a panel refresh), ending in a success page with a panel-true preview and summary. Progress (never the password) resumes via sessionStorage across reloads; "Skip setup" persists the latch and permanently restores stock behavior. New public endpoint `GET /api/setup/status` (`wizard`, `password_set`, `setup_completed`, `server_time`, `server_timezone`), new one-way settings latch `setup_completed` (accepted by `POST /update_settings`, survives heartbeats/triggers). Existing installations never see the wizard (any usage trace disables it; `?setup=1` is the only manual entry). The template gallery is now host-neutral (grid, pause criterion and select behavior are injectable; dashboard behavior unchanged) and gained `invalidatePreviews()` so a display-type change re-renders panel-true previews. New files: `server/static/js/setup-wizard.js`, `server/static/css/setup-wizard.css`, `server/internal/handlers/setup.go`.

- Nightly sleep window for panel care (server side, E5.3 / spec E5.2 phase 1): new settings `sleep_start`/`sleep_end` (`"HH:MM"` 24h wall-clock, both empty = off; persisted in `settings.json`, always present in `GET /settings`, validated by `POST /update_settings` — 400 on bad format, a resulting half-set window, or `start == end`; fields not sent stay unchanged). Inside the window the server suppresses interval refreshes (`should_refresh=false`); a manual trigger (`POST /api/trigger_refresh`) deliberately breaks through — an explicit user action wins. The window is evaluated against local server time (`TZ`), is half-open `[start, end)` and may wrap across midnight (e.g. `23:00`–`06:00`). Fail-open guarantees: first start (client never refreshed) refreshes even inside the window, and invalid values in a hand-edited `settings.json` disable the window with a warning instead of freezing the panel. Existing `settings.json` files without the fields load unchanged (window off). API-only for now (no settings UI yet).
- `GET /api/refresh_status` now reports why a refresh is requested: new field `reason` (`"manual"` for a trigger, `"interval"` for an elapsed interval, omitted when `should_refresh` is false; a trigger takes precedence when both apply). Groundwork for the client-side content skip (spec E5.2 phase 2) — old clients ignore the field, old servers simply don't send it.
- Client watchdog & recovery (E5.4): the client now survives driver/SPI errors and power outages without manual intervention. (1) **Driver recovery per cycle:** every exception from the Waveshare driver stack — driver load (`epdconfig` import side effects like `BadPinFactory`/`GPIOPinInUse`/`RuntimeError`, `OSError` from spidev), `init()`, `getbuffer()`, `display()`, and `init()` returning `-1` (previously ignored) — is logged with a full traceback (stable diagnosis line: `display recovery: driver reset after error`), followed by a full driver reset: best-effort `epdconfig.module_exit()` (panel power hard off — no more deep-sleep attempt over a possibly broken SPI bus) and re-instantiation of the `EPD()` object from the cached module on the next poll cycle (never `importlib.reload`, which would collide with the gpiozero pins). A failed cycle sends no heartbeat and keeps the content-skip hash unchanged (E5.2 contract). An `ImportError` still means permanent preview-only mode (no recovery, no escalation). (2) **Escalation to systemd:** after `EINK_HW_FAILURE_LIMIT` (new env var, default `3`, `0` = never) consecutive hardware failure cycles the client logs `too many consecutive display failures` (CRITICAL) and exits with code 1 — `Restart=always` + `RestartSec=10` in the unchanged systemd unit bring up a fresh process with a freshly imported driver stack. A successful physical panel write resets the counter; content skips, network errors and preview-only mode never touch it. (3) **Power-outage recovery:** a failed startup refresh (e.g. the server is still booting) is now retried on every poll cycle — regardless of `should_refresh` — until the first success, closing the gap where the panel could stay stale for up to a full `refresh_interval`; an unreachable server never touches the panel (no flicker) and never escalates. `scripts/test-setup.sh` now freezes the systemd watchdog contract (`RestartSec=10`/`5`, `After=`/`Requires=eink-server.service`). Known gap (documented): a driver hang in the busy-pin poll loop (`ReadBusy`, e.g. panel cable pulled) is not detected — a future sd_notify/`WatchdogSec` follow-up would cover that.
- Client content skip / panel care (E5.2, client phase): the client hashes the raw `GET /preview` PNG bytes (SHA-256) and skips the physical panel write when the server reports an interval refresh (`reason: "interval"`) and the bytes are identical to the last image successfully written to the panel — no `init`/`display`/`sleep` (the panel stays in deep sleep), the last-sent debug artifact is left untouched, and the heartbeat is still sent with `status: "skipped"` (stable log line: `skipping panel refresh (content unchanged)`). Manual triggers and refresh-status responses without a `reason` field (older servers) always write, so server and client stay independently deployable; designs with clock/minute widgets change bytes every minute, so the skip inherently never applies there (use the night window instead). The hash is recorded only after a successful panel write and lives in process memory only — a client restart always writes the first frame. New env vars: `EINK_CONTENT_SKIP` (default enabled; only the string `false`, case-insensitive, disables it) and `EINK_MAX_SKIP_HOURS` (default `24`, `0` = off): a panel-care guard that forces a write when the last real panel write is older (Waveshare recommends at least 1 refresh per 24 h; measured with `time.monotonic`). Documented meaning shift: `last_client_refresh` on the server now means "content current on the client", not "panel physically written" — the heartbeat `status` value distinguishes the two.

- Template gallery in the design dashboard: a new "Templates" section ships 8 ready-made 800x480 designs (Weather Dashboard, Family Calendar, Photo + Clock, Week Planner, News Briefing, Minimal Clock, Countdown, System Monitor) as static JSONs under `server/static/templates/` (embedded via the existing `go:embed`, no new endpoint). Every card shows a **panel-true, server-rendered live preview** (`POST /api/preview_live`, quantized to the configured display's palette, loaded strictly sequentially — never more than one render in flight — and cached per page session; a busy renderer falls back to a text placeholder without auto-retry). Clicking a card creates a copy as a new **inactive** design (collision-safe name suffix " 2", " 3", …) and opens it in the designer — activation stays a deliberate user click. The Countdown template's `__NEXT_NEW_YEAR__` token is resolved to Jan 1st of the next year on use; the Photo + Clock template picks up the newest Media Library image automatically (or keeps its designed placeholder when none is uploaded, badge "SETUP" marks templates that need user input). All templates use only exact driver palette colors, so 6-color panels render them pixel-sharp and the B/W panel stays contrast-readable. New file: `server/static/js/template-gallery.js`.

- Designer preview modal now shows the true panel rendering by default: `POST /api/preview_live` without `raw=true` returns the quantized image (dithering against the calibrated panel palette, output restricted to the active display's driver colors — 6 on epd7in3e, 2 on epd7in5_V2). A "Panel | Original" toggle in the modal switches to the full-color composition (`raw=true`) and back. Per modal session each mode is rendered at most once (object-URL cache, in-flight guard with disabled tabs and a "Rendering…" status line); a busy renderer (HTTP 503 from the render semaphore) shows "Renderer busy — try again" without auto-retry, and a failed initial render falls back to the saved design (`GET /preview`, now consistently quantized like the panel default). Fixed the preview object-URL leak: cached URLs are revoked when the modal closes. Note for future wiring of the mini live preview (currently a stub): debounce >= 500 ms, panel mode (no `raw`), latest-wins in-flight guard via AbortController — see `specs/E3.6-panel-palette-preview.md`.
- One-command installation: `curl -fsSL https://raw.githubusercontent.com/Kilian-Schwarz/E-INK-Picture/main/install.sh | bash` bootstraps a full native install (clone, dependencies, SPI, systemd). `setup.sh` is now headless-capable (`--headless`/`--yes`, auto-detected when piped), idempotent, supports `--update`, pins the Waveshare e-Paper library to a fixed commit (fetched sparsely instead of a full clone), fails loudly instead of silently falling back to preview-only on a Pi, aborts when a Docker install is already running, and renders systemd units (now `Restart=always`) from the `systemd/` templates. New `--dry-run` mode and `scripts/test-setup.sh` make the installer testable without hardware.
- **Authentication for the LAN-reachable server** (single admin password + client token, deny-by-default):
  - One admin password, bcrypt-hashed (cost 10) in `data/auth.json` (mode 0600, atomic writes) — never stored in `settings.json` or the repo. Set it via `POST /api/auth/setup` (only available while no password exists, then 403) or once at startup via the new `EINK_ADMIN_PASSWORD` env var (ignored with a warning once a password is set; clear the var after first start).
  - Session-cookie login for the browser: new `/login` page, `POST /api/auth/login`, `POST /api/auth/logout`, `GET /api/auth/status`. Cookie `eink_session` is HttpOnly, SameSite=Lax, 7-day sliding TTL, in-memory store bounded to 32 sessions (server restart = re-login). `EINK_COOKIE_SECURE=true` marks the cookie Secure for TLS-proxy setups.
  - Designer auth UI (new `server/static/js/auth.js`, loaded first): a global fetch interceptor redirects to `/login` when a same-origin API call returns 401 (expired session — no more silently failing saves); while no admin password is set, a dismissible warning banner offers an in-page setup dialog (password + confirmation, signs you in right after); when signed in, a Logout button appears in the topbar (moves into the burger menu on compact viewports).
  - New env var `EINK_CLIENT_TOKEN` (generated automatically into `.env` by `setup.sh`/`scripts/setup-local.sh` — also appended when updating an older install, never overwriting an existing value; manually: `openssl rand -hex 32`): the headless e-ink client authenticates via the `X-Client-Token` header on exactly its four endpoints (`GET /settings`, `GET /preview`, `GET /api/refresh_status`, `POST /api/client_heartbeat`). The token is no general key — every other route requires a session.
  - Deny-by-default guard middleware in front of the router: everything except `GET /health`, `GET /login`, the auth API and `/static/*` requires authentication — including future routes and unregistered paths (401 instead of 404). Page routes (`/`, `/designer`, `/media`) redirect to `/login`.
  - CSRF protection for cookie-authenticated mutating requests: SameSite=Lax plus a server-side Origin/`Sec-Fetch-Site` check (403 on cross-origin); token-authenticated client requests are exempt (no cookie, no CSRF vector).
  - Login and setup are rate-limited per IP (5 attempts / 60 s, then 429 with `Retry-After`).
  - **Upgrade path — no lockout:** existing installations stay open (exact current behavior) until a password is set; the server logs a loud warning on startup and hourly. **Recovery:** delete `data/auth.json` and restart (device access required) — the server returns to the open no-password state.
  - Reality check (documented, unchanged): the server speaks plain HTTP in the LAN; without TLS the session cookie is readable on the wire. TLS termination stays out of scope.
- Smart alignment guides in the designer (Canva/Figma-style): while moving an object, thin accent lines appear when its bounding-box edges or centers come within ~6 screen px (zoom-independent) of another object's edges/centers or the canvas edges/center, and the object snaps exactly onto the line. Resizing shows the same lines as visual feedback but does not snap (deliberate). Coexistence with the grid snap: within the threshold the guide wins (evaluated on the raw pointer position, so a target 2 px away snaps flush even if the grid just pulled the object away), outside it the grid behaves exactly as before. A new "Guides" toggle button next to the Grid button disables the feature for the session (default on, deliberately not persisted). Guides disappear immediately when a second finger starts a pinch (E3.2) and never create canvas objects (no layers-panel or live-preview churn). New file: `server/static/js/smart-guides.js`.

- Touch gestures on the designer canvas: pinch-zoom around the gesture midpoint and two-finger panning as a single gesture (zoom clamped to the existing 25-400% range, zoom label follows live; panning scrolls the canvas area), plus a long-press (500 ms) context menu on canvas objects with Duplicate / Bring to Front / Send to Back / Delete (mapped onto the existing toolbar actions, undo-able via history). A second finger cleanly aborts a running single-finger transformation (object snaps back, no history entry); single-finger interaction remains fully Fabric (select/drag/resize/rotate). Desktop mouse behavior is unchanged. New file: `server/static/js/touch-gestures.js`.

- Responsive designer layout for phones and tablets: below 768px the side panels become bottom sheets behind a fixed tab bar (Widgets | Layers | Properties, one sheet at a time, no backdrop — the uncovered canvas stays interactive), overflow topbar actions move into a burger menu, and a zoom-fit scales the full 800x480 design into the visible canvas area on load, resize, and orientation change. Between 768px and 1024px both panels collapse to 44px icon rails whose 260px flyouts overlay the canvas without reflowing it. Desktop (>=1024px) is pixel- and behavior-identical to before. New files: `server/static/css/responsive.css`, `server/static/js/responsive-layout.js`.
- New env var `EINK_MAX_CONCURRENT_RENDERS` (int >= 1, default `1`): global render semaphore — at most N preview renders run concurrently, additional requests wait in a FIFO queue and abort with HTTP 503 when the client disconnects (or the 30 s write timeout hits). `/health`, statics and all non-render endpoints are not gated. Invalid values log a warning and fall back to the default.
- New env var `EINK_GOMEMLIMIT` (default `64MiB`): soft memory limit for the Go runtime (`debug.SetMemoryLimit`), so free heap returns to the OS promptly after render bursts instead of growing to ~2x the live peak at GOGC=100. Format: positive integer with optional `MiB` suffix (bare number = bytes); **escape hatch** `off` (or `0`) disables the limit entirely. Precedence: `EINK_GOMEMLIMIT` > native `GOMEMLIMIT` env var (honored by the runtime itself, never overridden by the server) > 64 MiB default. A too-low limit shows up as high CPU/slow renders (GC thrash), never as an OOM kill — photo-heavy designs may transiently exceed the soft limit by design. Outlook: after the buffer-diet savings prove out on hardware, `48MiB` is a sensible follow-up tuning value.
- Explicit rename button (`#design-name-edit-btn`) in the designer topbar next to the design name — a touch-friendly alternative to the double-click rename, which remains available for desktop use.
- New settings `dither_algorithm` (`floyd_steinberg` | `atkinson`, default `floyd_steinberg`) and `calibration` (`default` | `off`, default `default`), persisted in `settings.json`, exposed via `GET /settings` and validated by `POST /update_settings` (unknown values return 400). Configuration is API-only for now (no settings UI yet).
- Atkinson dithering as an alternative error-diffusion algorithm (integer-only, 6 neighbors at 1/8 weight each — higher local contrast and brighter highlights, often a calmer image on e-ink).
- Panel calibration profiles (`server/internal/models/calibration.go`): a perceptual Spectra-6 panel palette (community-measured appearance values, white-point adapted) plus a per-profile gamma/saturation/contrast precompensation preset. Values are an initial guess and tunable at the physical panel without format changes.
- Deterministic calibration test design (`server/internal/services/testdata/designs/calibration.json`): 6 driver-color swatches, a 16-step gray ramp, skin-tone and mixed-color patches, and a text matrix for on-panel A/B evaluation via `POST /api/designs` → activate → `POST /api/trigger_refresh`.
- Client writes the exact panel image (after resize and mode conversion, immediately before the EPD driver call) to `/tmp/eink_last_sent.png` for hardware-in-the-loop verification. Path is configurable via `EINK_LAST_SENT_PATH`. The file is written atomically; a write failure never aborts the display refresh.
- Golden-file render test harness with byte-exact golden comparisons and exact palette assertions for both display profiles (`waveshare_7in3_e`, `waveshare_7in5_v2`). Update golden files with `go test ./internal/services -run TestGoldenRender -update`.
- `EINK_DISPLAY_TYPE` environment variable to configure the server's default display profile. Valid values: `waveshare_7in3_e` (6-color, driver `epd7in3e`) and `waveshare_7in5_v2` (B/W, driver `epd7in5_V2`). Applies only when `settings.json` has no `display_type` value.

### Changed

- **BREAKING (cloud mode): CORS wildcard removed.** With credentials (session cookies) `Access-Control-Allow-Origin: *` is forbidden. Local mode now sends no CORS headers at all (the designer is served same-origin); cloud mode requires an explicit comma-separated origin list in `CORS_ALLOWED_ORIGINS` (exact echo + `Access-Control-Allow-Credentials` + `Vary: Origin`). A configured `*` is rejected with a startup warning and treated as unconfigured (same-origin only). The default for `CORS_ALLOWED_ORIGINS` changed from `*` to empty. Preflights now allow `PUT`/`PATCH`/`DELETE` and the `X-Client-Token` header.

- Render memory diet (byte-identical output, verified by the golden-file suite): the panel-calibration precompensation pass now runs in place on the render-private RGBA buffer (saves a full-canvas copy, ~1.5 MB per render), and the supersampling downscale reuses a pooled CatmullRom scaler (`draw.Kernel.NewScaler`, bounded cache of 3 geometries) instead of allocating a ~24.6 MB temp buffer per render. A warm high-quality render of the basic test design now allocates ~11.4 MB (before: ~38.2 MB), enforced by `TestRenderAllocBudget` (< 20 MiB).
- Docker memory limits documented as ineffective on standard Raspberry Pi OS: `deploy.resources.limits.memory` requires the cgroup memory controller, which is inactive by default on the Pi (hardware finding). See INSTALL.md — the new in-process limits (`EINK_GOMEMLIMIT`, `EINK_MAX_CONCURRENT_RENDERS`) are the effective mechanism; no cgroup fix is shipped.
- The designer is now operable by touch and pen input: the Fabric.js canvas runs in Pointer Events mode (`enablePointerEvents`, with feature detection), so select/drag/resize/rotate use one unified input path for mouse, touch, and stylus. The crop dialog was migrated from mouse events to pointer events (pointer capture, `pointercancel` handling, `touch-action: none` on the crop area), making crop selection draggable and drawable by finger. Desktop mouse behavior is unchanged.
- **Calibrated dithering is now the default for the 6-color panel (`waveshare_7in3_e`):** error diffusion runs against the measured panel appearance of the six Spectra-6 colors (plus a mild saturation precompensation) and maps the result index-preserving back onto the pure driver colors. The PNG sent to the client still contains exclusively the 6 ideal driver colors — client and driver are unchanged — but the rendered dithering pattern (and thus the panel image) changes visibly. **Escape hatch:** `POST /update_settings` with `{"calibration":"off"}` restores the previous output byte-exactly. The B/W profile (`waveshare_7in5_v2`) uses an identity profile and is completely unaffected.
- The Go renderer now honors `Element.Rotation` (degrees, clockwise, around the element's top-left anchor — matching the Fabric.js designer semantics). Rotated elements render correctly on the panel and in the server-rendered live previews; anti-aliased rotated edges are dithered before palette quantization. Exact multiples of 90° stay pixel-exact axis-parallel, and element culling accounts for the rotated bounding box.
- Fresh installations (no `settings.json` and no `EINK_DISPLAY_TYPE` set) now default to `waveshare_7in3_e` (7.3" 6-color) instead of `waveshare_7in5_v2` (7.5" B/W). This matches the client's default driver `epd7in3e`. **Migration note:** users of the 7.5" V2 panel without a persisted `settings.json` must set `EINK_DISPLAY_TYPE=waveshare_7in5_v2` or select the display once in the settings UI. Existing `settings.json` values always win and are not affected.

### Removed

- Env var `SECRET_KEY` and `Config.SecretKey` (dead code, never referenced): server-side random session tokens need no signing secret. Remove the variable from existing `.env` files; it is ignored.

### Fixed

- A canvas zoomed beyond the visible area is now fully reachable by scrolling: the canvas area no longer flex-centers its overflow, which used to make the left half of the overhang structurally unscrollable (pre-existing on desktop, exposed by pinch-zoom).
- Touch handles no longer cover small objects' centers: Fabric's 24 px touch hit area for the selection handles is now clamped per object to a third of the on-screen size (12-24 px), so dragging a default 200x60 text element at mobile fit-zoom moves it instead of accidentally scaling its height (E3.3 verification finding). Handle rendering and mouse behavior are unchanged.
- Server font handling is now safe under concurrent renders: the font cache stores the parsed font instead of `font.Face` instances (which are not concurrency-safe and could race up to a panic when `EINK_MAX_CONCURRENT_RENDERS` >= 2); a fresh face is created per use.
- The e-ink client resizes preview images with NEAREST instead of LANCZOS on a size mismatch, so the server's palette-exact dithering can never be interpolated into mixed colors; a warning with both sizes flags the misconfiguration.
